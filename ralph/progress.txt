## Codebase Patterns
- Project root is `agent-observatory/` (not `ralph/`); all code/docs changes go there
- Branch created from `main` in the `agentew_valley` monorepo root
- Typecheck: `pnpm --filter @agent-observatory/server build` (tsc); web package has a pre-existing vite config TS error unrelated to migration work
- docs/migration/ is created fresh; no prior content exists
- Migration governance docs should prefer measurable acceptance tables (`Target`, `Measurement Source`, `Alert Threshold`) to keep criteria verifiable
- Ops-domain schema guidance should enforce shared audit/tenant columns (`workspace_id`, `created_at`, `updated_at`, `actor`) and keep UAEP telemetry storage boundary separate from operational source-of-truth tables
- Backfill planning docs should define per-entity source->target mappings together with field-level transform/null rules and explicit skip-code taxonomy for unsupported records
- Backfill idempotency policy should use operation-aware keys (`<entity>:<workspace_id>:<entity_id>:<operation>:<version_token>`) and deterministic replay sorting (`version_token`, `source_sequence`, operation rank, primary id) so reruns are reproducible
- Migration rehearsal validation consumes CSV artifacts with fixed headers (`entity,source_count,target_count` and `entity,diff_count`) and enforces required entities: `tasks`, `reviews`, `notifications`, `activities`, `webhooks`
- On pnpm v10, run `pnpm approve-builds` and approve `better-sqlite3` + `esbuild` before server tests; otherwise native binding load errors occur
- Shadow comparator output should keep status values fixed (`match`, `mismatch`, `missing_legacy`, `missing_new`) and use deterministic `$.path` field diffs with sorted object keys for stable reporting
